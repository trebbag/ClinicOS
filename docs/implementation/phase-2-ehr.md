# Phase 2: EHR integration

Phase 2 should only begin once phase 1 is stable.

## Preconditions
- role-based access model exists
- audit logging is complete
- approved documents are under control
- scorecards run deterministically
- approval boundaries are tested
- PHI boundary document is complete

## Phase 2 candidate capabilities
- controlled Athena exports via service account
- PHI-safe ingestion layer
- chart-audit sampling workflow
- inbox/test-result governance workflows
- role-based access to patient-level dashboards
